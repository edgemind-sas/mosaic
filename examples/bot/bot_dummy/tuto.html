1. Introduction

Cette page présente la création d'un bot de trading en utilisant la librairie MOSAIC. Rappelons que la librairie MOSAIC est disponible sur GitHub et offre des fonctionnalités pour élaborer des stratégies de trading, basées sur divers indicateurs techniques.

2. Initialisation de l'environnement

2.1. Prérequis techniques

La librairie MOSAIC nécessite une version de Python >= 3.10.

Afin de faciliter vos projets de développement, il est recommandé d'utiliser un environnement virtuel Python pour réaliser ce tutoriel. Commencez donc par installer le gestionnaire pew si cela n'est pas déjà fait : 

pip install pew

Créez ensuite un nouvel environnement virtuel : 

pew new mosaic_bot_dummy

Créez à présent un répertoire qui contiendra les fichiers de ce tutoriel, e.g. bot_dummy, et déplacez-vous dans ce répertoire : 

mkdir bot_dummy
cd bot_dummy

À noter que vous pouvez sauvegarder ce répertoire comme étant associé à l'environnement virtuel que vous venez de créer avec la commande : 

pew setproject

Installez enfin la librairie MOSAIC dans votre environnement à partir de github : 

pip install https://github.com/edgemind-sas/mosaic.git

Si l'installation a marché, vous devriez être capable d'importer la librairie MOSAIC dans une session Python : 

import mosaic

3. Création d'un bot

3.1. Spécification du bot

3.1.1. Configuration YAML

Pour définir un bot, le plus simple est de créer un fichier de configuration permettant de spécifier les caractéristiques du bot.

Pour ce faire, créez un fichier YAML bot.yaml contenant les éléments suivants : 

# Indicator definitions
indics:
  rsi_short: &rsi_short
    cls: RSI
    length: 15

  mfi_short: &mfi_short
    cls: MFI
    length: 60

  mfi_long: &mfi_long
    cls: MFI
    length: 120

# Definition of return prediction model.
# Here we define a Logit up/down prediction model.
predict_models:
  logit: &pm_logit
    cls: PMLogit
    returns_horizon: 60
    features:
      - *mfi_short
      - *mfi_long

# Bot specification
bot:
  name: "bot_dummy"
  mode: "btfast"
  bt_buy_on: high
  bt_sell_on: low
  
  ds_trading: &ds_trading
    symbol: "BTC/FDUSD"
    timeframe: "1s"
    dt_start: "2023-09-15 00:00:00"
    dt_end: "2023-09-16 00:00:00"

  ds_fit:
    <<: *ds_trading
    dt_start: "2023-09-12 00:00:00"
    dt_end: "2023-09-15 00:00:00"

  portfolio: 
    quote_amount_init: 1
    
  decision_model:
    cls: DM2ML
    buy_threshold: 0
    sell_threshold: 0
    pm_buy:
      <<: *pm_logit
      direction: up

    pm_sell:
      <<: *pm_logit
      direction: down

  order_model:
    cls: OrderMarket
    
  invest_model:
    cls: InvestLongModel
    nb_buy_allowed: 1
    buy_quote_rate: 1
    sell_base_rate: 1
        
  exchange :
    cls: ExchangeCCXT
    name: binance
    fees_rates:
      taker: 0
      maker: 0

3.1.2. Éléments de Configuration d'un Bot

Définition des indicateurs

  • indics: Ce bloc définit plusieurs indicateurs techniques utilisés par le modèle de décision.
    • rsi_short: Définit un indicateur RSI avec une fenêtre de 15 unités de temps.
    • mfi_short: Définit un indicateur MFI avec une fenêtre de 60 unités de temps.
    • mfi_long: Définit un autre indicateur MFI, mais avec une fenêtre de 120 unités de temps.

Modèles de Prédiction

  • predict_models: Bloc décrivant les modèles de prédiction des rendements.
    • logit: Utilise un modèle Logit pour la prédiction à la hausse/baisse des rendements futurs. Utilise mfi_short et mfi_long comme variables explicatives.

Spécification du Bot

  • bot: Ce bloc spécifie les détails de configuration du bot de trading.

    • name: Nom du bot, "logit" en l'occurrence.
    • mode: Mode de fonctionnement, "btfast" pour un backtesting rapide.
    • bt_buy_on et bt_sell_on: Hypothèses pour les prix d'achat et de vente en backtesting.

    Configuration des Sources de Données

    • ds_trading: Source de données pour le trading en temps réel ou en backtesting.
    • ds_fit: Source de données pour l'ajustement du modèle.

    Modèles

    • decision_model: Spécifie le modèle de décision pour les ordres d'achat et de vente.
    • order_model: Spécifie le modèle d'ordre, ici un ordre au marché.
    • invest_model: Spécifie le modèle d'investissement, autorisant un seul achat et des taux de vente/achat fixes.

    Échange

    • exchange: Détails de l'échange où les données OHLCV sont récupérées.
      • name: Le nom de l'échange, ici Binance.
      • fees_rates: Frais de transaction utilisés.

3.2. Exécution du bot en mode backtest

Créez ensuite le fichier Python bot.py suivant permettant de backtester le bot défini dans le fichier YAML précédent. 

import mosaic.trading as mtr
import yaml

with open("bot.yaml", 'r', encoding="utf-8") as yaml_file:
    bot_config = yaml.load(yaml_file,
                           Loader=yaml.SafeLoader)
    bot_config["bot"].setdefault("cls", "BotTrading")
    bot = mtr.BotTrading.from_dict(bot_config["bot"])

bot.start(
    data_dir=".",
    progress_mode=True,
)

print(bot.portfolio)

Lancez le bot en exécutant la commande python bot.py. À la fin de l'exécution, vous devriez observer l'état du portefeuille (portfolio) du bot à la fin du backtest. Dans ce cas, nous ne sommes pas rentables…
